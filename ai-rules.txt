АРХИТЕКТУРНЫЕ ПРАВИЛА GAME CENTER:

- vi3na1bita-games — это изолированный micro-app для iframe внутри основного приложения vi3na1bita-music.
- Основное приложение остаётся владельцем плеера, авторизации, профиля, статистики, достижений, backup/sync.
- Game Center не имеет права вызывать play, pause, stop, mute, volume, seek, next, prev.
- Game Center не должен создавать AudioContext, WebAudio, Howler, <audio> или игровые звуки без отдельного parent-mediated слоя.
- Game Center не должен читать OAuth token, localStorage, IndexedDB, metaDB или parent.window.
- Любая связь с основным приложением — только через postMessage bridge.
- Разрешённые исходящие сообщения: GC_READY, GC_REQUEST_SNAPSHOT, GC_DOOR_CLICKED, GC_CLOSE и будущие write-limited события только после валидации parent-side.
- Запрещённые исходящие сообщения: любые команды управления музыкой, громкостью, плейлистом, избранным, качеством.
- Snapshot от parent считается read-only.
- При standalone-запуске приложение должно работать как preview без parent bridge.
- Ошибка Game Center не должна ломать основное приложение и не должна мешать воспроизведению музыки.
- Для iOS приоритет: не мешать фоновому воспроизведению и lockscreen playback основного приложения.
- Первая визуальная стратегия: CSS/2.5D panorama без WebGL и без heavy runtime.
- Любой RAF/timer должен останавливаться или деградировать при document.hidden.
- Не добавлять service worker в game repo на раннем этапе, чтобы не конфликтовать с root SW основного приложения.
- Не создавать папку Games/ внутри этого репозитория: корень репозитория деплоится в bucket prefix /Games/.
